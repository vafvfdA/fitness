# 健身饮食应用 PRD

## 1. 产品概述

建设一个移动端优先的健身和饮食管理应用，既能满足个人日常使用，也能作为 Java 后端架构和编码练习项目。优先形态是微信小程序，后端使用 Java 技术栈。

应用帮助用户记录每日训练、饮食、热量、身体指标、计划和知识库问答。

## 2. 目标用户

主要用户画像：

- Java 开发工程师，希望做一个真实可用的个人项目。
- 主要在手机上记录训练和饮食。
- 希望快速记录每天练了什么、吃了什么。
- 希望项目能锻炼后端架构、领域建模、接口设计、测试和可维护性。

## 3. 产品目标

1. 记录每日训练动作、组数和预估消耗热量。
2. 记录每日饮食和预估摄入热量。
3. 通过紧凑日历查看每天训练内容和热量情况。
4. 管理个人身体指标和目标体重。
5. 配置健身和饮食计划，并支持提醒。
6. 维护运动、食材、营养相关知识库。
7. 基于知识库提供流式问答。

## 4. 非目标

- MVP 不做医疗诊断。
- 不替代专业营养师、医生或康复师建议。
- 第一阶段不做公开社交平台。
- 第一阶段不做商业化 SaaS。
- 第一阶段不接入手环、手表等穿戴设备。

## 5. 核心页面

### 5.1 训练页面 A

用途：

- 查看选定日期练了什么。
- 记录动作、组数、次数、重量、时长和预估消耗热量。
- 能快速跳转到同一天的饮食页面 B。

主要内容：

- 日期选择。
- 部位标签：胸、肩、背、腿、臂、核心、有氧、其他。
- 训练汇总：动作数量、总组数、预估消耗热量。
- 动作列表。
- 从模板添加。
- 自定义动作。
- 保存草稿或提交。

关键交互：

- 用户可以选择训练模板，生成当天动作列表。
- 用户可以编辑组数、次数、重量、时长和热量。
- 用户可以跳到饮食页面 B，并保留当前日期。

### 5.2 饮食页面 B

用途：

- 查看选定日期吃了什么。
- 记录食物、餐次、数量和预估热量。
- 计算每日摄入热量。

主要内容：

- 日期选择。
- 餐次分区：早餐、午餐、晚餐、加餐。
- 食物列表。
- 当日摄入总热量。
- 从食物模板添加。
- 自定义食物。
- 可选的热量估算辅助。

关键交互：

- 用户可以选择常用食物模板。
- 用户输入数量后计算热量。
- 用户可以回到同一天的训练页面 A。

### 5.3 日历页面

用途：

- 以月份视角查看训练和热量概览。

每天格子展示精简内容：

- 主要训练部位，例如胸、肩、背、腿、臂。
- 预估训练消耗热量。
- 如果空间允许，可以展示摄入热量或热量差。

交互：

- 点击某天进入当天训练和饮食详情。
- 切换月份。
- 高亮今天。
- 标识休息日。

### 5.4 知识库页面

用途：

- 维护运动、食物、热量、营养、训练计划相关知识。
- 基于已存知识进行问答。
- 回答要求支持流式输出。

主要内容：

- 知识条目列表。
- 新增、编辑、删除知识条目。
- 知识分类：训练、食物、营养、计划、风险提示、其他。
- 问答输入框。
- 流式回答区域。

约束：

- 回答不得宣称医疗确定性。
- 尽量引用或展示命中的知识来源。
- 后端应支持流式响应；小程序端如受平台限制，需要提供降级方案。

### 5.5 计划页面

用途：

- 设置当前身体状态、目标体重、训练计划和饮食目标。
- 展示距离目标还差多少。
- 根据计划提供提醒。

主要内容：

- 当前体重。
- 目标体重。
- 距离目标体重差值。
- 计划类型，例如练三休一。
- 部位轮换，例如胸、肩、背。
- 饮食目标，例如每日热量预算。
- 提醒设置。

交互：

- 用户可以更新当前体重。
- 用户可以设置目标体重。
- 用户可以配置训练周期。
- 用户可以看到今日计划训练内容。
- 如果开启提醒，系统按计划提醒。

## 6. 功能模块

1. 用户和个人资料模块。
2. 身体指标模块。
3. 训练记录模块。
4. 训练模板模块。
5. 饮食记录模块。
6. 食物模板模块。
7. 日历汇总模块。
8. 计划和提醒模块。
9. 知识库模块。
10. 流式问答模块。
11. 统计分析模块。

## 7. MVP 范围

MVP 优先保证每日记录闭环：

1. 微信登录或开发模式用户身份。
2. 训练页面 A。
3. 饮食页面 B。
4. 日历页面。
5. 基础计划页面，包含当前体重、目标体重和训练周期。
6. 训练和食物基础模板。
7. 后端 CRUD API。
8. 数据库表结构和迁移。
9. 基础自动化测试。

知识库流式问答可以在记录闭环稳定后实现，除非用户明确指定优先开发。

## 8. 成功标准

- 用户可以在 2 分钟内用模板记录一次完整训练。
- 用户可以在 1 分钟内用模板记录一顿常见饮食。
- 日历能清晰展示当月训练状态。
- 用户能看到当前体重、目标体重和差值。
- 后端数据按用户隔离。
- 项目结构足够清晰，便于后续扩展。
