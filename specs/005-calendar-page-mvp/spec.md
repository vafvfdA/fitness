# Feature Specification: 训练日历页面联调

**Feature Branch**: `005-calendar-page-mvp`

**Created**: 2026-07-21

**Status**: Draft

**Input**: 用户确认 004 已验收并提交，下一步开发训练日历页面联调。

## User Scenarios & Testing

### User Story 1 - 查看月度训练日历 (Priority: P1)

用户进入日历页后，可以看到当前月份的日历格子。每个日期格子展示日期、训练部位摘要和当天消耗热量。

**Why this priority**: 日历页是原始需求中的核心页面，用来快速回看每天练了什么。

**Independent Test**: 后端返回某月训练摘要时，页面能把摘要映射到对应日期格子；无训练日期显示空状态。

**Acceptance Scenarios**:

1. **Given** 用户打开日历页，**When** 页面加载，**Then** 系统展示当前月份日历。
2. **Given** 某天有训练记录，**When** 页面渲染该日期格子，**Then** 格子显示训练部位和消耗热量。
3. **Given** 某天没有训练记录，**When** 页面渲染该日期格子，**Then** 格子只显示日期。

---

### User Story 2 - 切换月份 (Priority: P2)

用户可以切换上一个月和下一个月，查看不同月份的训练摘要。

**Why this priority**: 日历页需要支持回看历史训练，单月固定展示不足以满足管理需求。

**Independent Test**: 点击上月或下月后，页面请求对应月份并刷新日历格子。

**Acceptance Scenarios**:

1. **Given** 用户在当前月，**When** 点击上月，**Then** 页面展示上一个月。
2. **Given** 用户在当前月，**When** 点击下月，**Then** 页面展示下一个月。

---

### User Story 3 - 从日期进入记录页 (Priority: P3)

用户点击某个日期后，可以进入该日期的训练记录页，也可以从日历页进入同日饮食页。

**Why this priority**: 日历不只是展示，还应该作为日期导航入口。

**Independent Test**: 点击日期后，训练页收到相同日期参数；点击饮食入口后，饮食页收到相同日期参数。

**Acceptance Scenarios**:

1. **Given** 用户点击某天，**When** 页面跳转，**Then** 训练页打开并保留该日期。
2. **Given** 用户点击某天的饮食入口，**When** 页面跳转，**Then** 饮食页打开并保留该日期。

### Edge Cases

- 后端请求失败时，页面必须显示错误提示，不应白屏。
- 月份切换跨年时必须正确处理，例如 2026-01 上月为 2025-12。
- 日历首尾需要补齐空格子，保证周布局稳定。
- 后端返回空 `days` 时，月历仍正常展示。

## Requirements

### Functional Requirements

- **FR-001**: 日历页必须调用月度训练日历接口。
- **FR-002**: 日历页必须展示当前月份、周标题和日期格子。
- **FR-003**: 有训练的日期必须展示训练部位摘要。
- **FR-004**: 有训练的日期必须展示当天消耗热量。
- **FR-005**: 页面必须支持切换上月和下月。
- **FR-006**: 页面必须支持点击日期进入训练页。
- **FR-007**: 页面必须支持从日期进入饮食页。
- **FR-008**: 页面必须展示月度训练天数和总消耗。
- **FR-009**: 页面必须处理加载、空数据和失败状态。

### Key Entities

- **月历格子**: 表示日历中的一个格子，可能为空，也可能对应一个日期。
- **训练日摘要**: 某日期的训练部位、组数、消耗热量和训练记录数量。
- **月度摘要**: 当前月的训练天数和总消耗。

## Success Criteria

### Measurable Outcomes

- **SC-001**: 用户可以在 1 个页面内查看整月训练部位和热量消耗。
- **SC-002**: 用户可以通过 1 次点击切换到上月或下月。
- **SC-003**: 用户可以通过 1 次点击从日期进入对应训练页。
- **SC-004**: 日历格子构造逻辑有自动化测试覆盖。
- **SC-005**: 后端回归测试保持通过。

## Assumptions

- 后端月历接口已经存在，路径为 `GET /api/v1/workout-records/calendar?month=yyyy-MM`。
- 小程序仍使用开发期用户 ID `1001`。
- 本轮不实现饮食日历聚合，只提供进入同日饮食页入口。
