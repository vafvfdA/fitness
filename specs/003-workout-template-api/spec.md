# Feature Specification: 训练模板接口

**Feature Branch**: `003-workout-template-api`

**Created**: 2026-07-21

**Status**: Draft

**Input**: 用户确认下一个功能优先开发训练模板接口，用于支撑 A 界面快速录入训练动作、组数和消耗热量，并能从模板生成当天训练记录。

## User Scenarios & Testing

### User Story 1 - 创建训练模板 (Priority: P1)

用户可以为某个训练部位创建个人训练模板，模板包含模板名称、训练部位、描述和多个默认动作。每个动作可以设置默认组数、默认次数、默认重量、默认时长、排序和预估消耗热量。

**Why this priority**: 模板是后续快速录入训练记录的基础，没有模板就无法减少每日重复输入。

**Independent Test**: 创建一个“胸日模板”，包含卧推和上斜哑铃推举，系统返回模板详情，并能看到动作顺序和默认训练参数。

**Acceptance Scenarios**:

1. **Given** 用户提供模板名称、训练部位和动作列表，**When** 创建模板，**Then** 系统保存模板并返回模板详情。
2. **Given** 用户提交缺少模板名称或训练部位的请求，**When** 创建模板，**Then** 系统拒绝请求并返回校验错误。

---

### User Story 2 - 查询训练模板 (Priority: P2)

用户可以查看自己的训练模板列表，也可以按训练部位筛选，并进入模板详情查看每个动作的默认参数。

**Why this priority**: 小程序 A 界面需要先展示可选模板，用户才能快速选择练什么。

**Independent Test**: 用户创建胸、背两个模板后，按“胸”筛选只返回胸部模板，详情接口返回完整动作列表。

**Acceptance Scenarios**:

1. **Given** 用户已有多个模板，**When** 查询模板列表，**Then** 系统返回当前用户可用模板。
2. **Given** 用户按训练部位筛选，**When** 查询模板列表，**Then** 系统只返回对应部位模板。
3. **Given** 用户查看不属于自己的模板，**When** 查询详情，**Then** 系统不返回其他用户私有模板。

---

### User Story 3 - 从模板生成训练记录 (Priority: P3)

用户选择某个训练模板后，可以指定训练日期，一键生成当天训练记录草稿。系统根据模板动作和默认组数生成训练动作与组明细，用户后续可以在训练记录页面继续调整。

**Why this priority**: 这是模板功能对录入效率最直接的价值，能把“选模板”转化为真实训练记录。

**Independent Test**: 用户选择“胸日模板”并指定日期后，系统创建一条训练记录，动作和组数来自模板。

**Acceptance Scenarios**:

1. **Given** 用户已有一个包含两个动作的模板，**When** 从模板生成指定日期训练记录，**Then** 系统创建训练记录并返回完整训练记录详情。
2. **Given** 模板动作设置默认组数为 3，**When** 生成训练记录，**Then** 系统为该动作生成 3 组训练组明细，组号从 1 开始。
3. **Given** 用户尝试使用其他用户的私有模板，**When** 生成训练记录，**Then** 系统拒绝使用该模板。

### Edge Cases

- 模板动作列表为空时，创建请求必须失败，避免生成不可用模板。
- 模板动作默认组数必须大于 0，避免生成空训练记录。
- 未传训练日期时，从模板生成训练记录默认使用当前服务器日期。
- 用户只能访问自己的私有模板；系统模板保留为未来扩展，本轮只要求不破坏 `is_system` 字段兼容性。
- 从模板生成训练记录时，训练记录总组数由模板动作默认组数汇总计算。

## Requirements

### Functional Requirements

- **FR-001**: 系统必须允许用户创建个人训练模板。
- **FR-002**: 系统必须保存模板的名称、训练部位、描述和动作列表。
- **FR-003**: 系统必须校验模板名称、训练部位和动作名称不能为空。
- **FR-004**: 系统必须校验每个模板动作的默认组数大于 0。
- **FR-005**: 用户必须能够查询自己的训练模板列表，并可按训练部位筛选。
- **FR-006**: 用户必须能够查看某个模板的完整详情。
- **FR-007**: 系统必须按用户隔离私有模板，不能向用户返回其他用户的私有模板。
- **FR-008**: 用户必须能够从自己的模板生成一条训练记录。
- **FR-009**: 从模板生成训练记录时，系统必须把模板动作转换为训练动作，并按默认组数生成训练组。
- **FR-010**: 从模板生成训练记录后，返回结果必须与训练记录接口的记录详情结构一致，便于小程序直接跳转或展示。

### Key Entities

- **训练模板**: 用户用于快速录入训练的模板，包含名称、训练部位、描述、是否系统模板等信息。
- **训练模板动作**: 模板内的默认动作，包含动作名称、默认组数、默认次数、默认重量、默认时长、排序和预估消耗热量。
- **训练记录**: 从模板生成后的当天实际训练记录，复用已有训练记录模型。

## Success Criteria

### Measurable Outcomes

- **SC-001**: 用户可以在 1 次请求内创建包含多个动作的训练模板。
- **SC-002**: 用户可以在 1 次请求内从模板生成当天训练记录。
- **SC-003**: 查询模板详情时，动作顺序和默认参数与创建时一致。
- **SC-004**: 用户隔离场景被自动化测试覆盖，其他用户私有模板不会被读取或使用。
- **SC-005**: 模板生成训练记录后，总组数与模板动作默认组数之和一致。

## Assumptions

- 本轮仅实现个人私有模板；系统模板数据初始化留到后续功能。
- 当前登录体系未完成，接口继续使用 `X-User-Id` 识别用户。
- 模板生成训练记录时，训练记录标题默认使用模板名称。
- 模板动作的预估热量会汇总为训练记录预估热量。
- 本轮不实现模板编辑和删除，避免扩大范围。
