# Feature Specification: 今日训练饮食聚合接口

**Feature Branch**: `002-today-summary-api`

**Created**: 2026-07-21

**Status**: Draft

**Input**: 用户要求开始下一个功能。结合当前训练记录接口和饮食记录接口状态，下一个功能为今日页聚合接口。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 查看今日训练饮食总览 (Priority: P1)

用户打开今日页时，可以一次看到指定日期的训练次数、训练部位、总组数、运动消耗、饮食摄入和净热量。

**Why this priority**: 今日页是小程序最常用的入口。聚合接口能避免前端同时请求多个接口再自行拼接数据。

**Independent Test**: 为同一用户创建指定日期的训练记录和饮食记录后，请求今日聚合接口，响应包含训练汇总、饮食汇总和净热量。

**Acceptance Scenarios**:

1. **Given** 用户在 2026-07-21 录入 1 条训练和 1 条饮食，**When** 查询今日聚合，**Then** 返回训练、饮食和净热量汇总。
2. **Given** 用户当天训练消耗 320，饮食摄入 770，**When** 查询今日聚合，**Then** 净热量为 450。

---

### User Story 2 - 没有数据时返回空汇总 (Priority: P2)

用户在还没录入训练或饮食时打开今日页，也能看到结构稳定的 0 值汇总。

**Why this priority**: 空状态是首页高频状态，接口不能让前端为缺失字段写复杂兜底逻辑。

**Independent Test**: 查询没有任何训练和饮食记录的日期，响应中的训练和饮食数字均为 0，列表为空。

**Acceptance Scenarios**:

1. **Given** 用户当天没有记录，**When** 查询今日聚合，**Then** 返回 0 值训练、0 值饮食和净热量 0。

---

### User Story 3 - 用户数据隔离 (Priority: P3)

同一天其他用户的训练和饮食记录不会影响当前用户的今日聚合结果。

**Why this priority**: 健身和饮食数据属于敏感个人数据，必须按用户隔离。

**Independent Test**: 为两个用户创建同日数据，当前用户查询时只聚合自己的数据。

**Acceptance Scenarios**:

1. **Given** 用户 A 和用户 B 在同一天都有记录，**When** 用户 A 查询今日聚合，**Then** 响应不包含用户 B 的训练和饮食统计。

### Edge Cases

- 查询日期没有任何数据时返回完整结构和 0 值。
- 只有训练没有饮食时，饮食汇总为 0，净热量为负运动消耗。
- 只有饮食没有训练时，训练汇总为 0，净热量等于饮食摄入。
- 不同用户同一天的数据必须隔离。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须支持按日期查询当前用户的训练饮食聚合结果。
- **FR-002**: 系统必须返回训练汇总，包括训练次数、训练部位、总组数和估算运动消耗。
- **FR-003**: 系统必须返回饮食汇总，包括饮食记录数、食物条目数、总热量、蛋白质、脂肪和碳水。
- **FR-004**: 系统必须返回净热量，计算规则为饮食总热量减去训练估算消耗。
- **FR-005**: 系统必须在无数据日期返回稳定的 0 值结构。
- **FR-006**: 系统必须按用户隔离聚合结果。
- **FR-007**: 系统必须使用统一响应结构返回数据和错误。

### Key Entities *(include if feature involves data)*

- **今日聚合**: 指定用户在指定日期的训练、饮食和净热量总览。
- **训练汇总**: 来自训练记录表的训练次数、训练部位、组数和消耗。
- **饮食汇总**: 来自饮食记录和食物明细表的摄入热量和营养素合计。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 小程序今日页可以通过一次请求获得训练和饮食总览。
- **SC-002**: 无数据日期返回完整 0 值结构。
- **SC-003**: 聚合结果只包含当前用户指定日期的数据。
- **SC-004**: 自动化测试覆盖有数据、无数据、用户隔离和净热量计算。

## Assumptions

- 当前阶段不修改训练和饮食接口，只新增聚合查询接口。
- 当前阶段不引入缓存，直接基于数据库实时聚合。
- 当前阶段继续使用 `X-User-Id` 请求头作为开发期用户识别方式。
